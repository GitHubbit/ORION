neo4j ubergraph data comparison

Objective is to get some metrics on hits and misses with the current covid robokop DB (AUTOMAT uberongraph, "AMDB") 
and data parsed from ubergraph (properties-nonredundant.ttl, "PNR")

Obvious differences:
	the number of nodes/edges in each dataset (match (n) return count(*), match ()-[e]->() return count(*)):
		AMDB has 208725 nodes, 9259815 edges
		PNR has 110927 nodes, 2539226 edges (72181 nodes and 332972 edges did not normalize)

	the AMDB data does not appear to have had any edge or node normalizations.
	
	the AMDB data is not consistent on populating data elements (for example, 79465 nodes are missing a name). this is unfortunate as the name field appears to be being used for the node type (e.g. "organic ion", "hydroxides", etc.).
	
	the AMDB data has edge relationships (field relation) specified as CURIEs (e.g. RO:0002323, etc.) where PNR has used the biolink CURIE retrieved from edge normalization (e.g. biolink:related_to, etc.).
	
	the AMDG database has a number of invalid predicate IDs. (e.g. GAMMA:0, owl:ObjectProperty
	
test 1: Does AMDB have records that PNR omitted due to node normalizaton failures.
	query: match (n) where n.id in ["BFO:0000002", "BFO:0000004", "BFO:0000040", "CHEBI:24432", "BFO:0000020", "BFO:0000017", "CHEBI:52206", "CHEBI:50906", "BFO:0000023", "CHEBI:25212"]  return count(*)
	result: AMDB does not have these records either

test 2: Does AMDB have records that were created in PNR:
	query: match (n) where n.id in ["GO:1990324", "GO:0048743", "CHEBI:36758", "CHEBI:32661", "UBERON:0014738", "UBERON:0009216", "MONDO:0009855", "MONDO:0014165", "HP:0031604", "NCBITaxon:451871"]  return n.id
	result: "NCBITaxon:451871",  was not found in AMDB data, as a matter of fact AMDB does not appear to have any taxa nodes defined.
	
test 3: find the various node to node relationsships by node type.
	query: match (n1)-[e]-(n2) return distinct split(n1.id,':')[0], <edge relation column>, split(n2.id,':')[0], count(*) order by split(n1.id,':')[0], split(n2.id,':')[0], <edge relation column> limit 1000
